## What We Built

Completed a manual reliability pass across the sidebar file explorer, code editor command routing, and terminal clipboard behavior.

## Terminal Selection And Clipboard

Reworked terminal selection/copy handling to support modern CLI/TUI developer workflows.

Completed:

- Routed terminal copy, paste, and select-all before egui can consume terminal shortcuts.
- Preserved terminal selection after copy instead of clearing it immediately.
- Ignored modifier-only key events so pressing Command alone no longer clears the active terminal selection.
- Replaced app-side terminal selection with Alacritty-backed selection state.
- Rendered selection highlights from the emulator selection range.
- Extracted copied text from the emulator-selected grid range.
- Fixed forward, backward, and multiline drag selection boundaries.
- Fixed the case where copied output collapsed to the final selected character.
- Made normal drags inside mouse-reporting CLI/TUI apps become LLNZY/Alacritty selections after the pointer leaves the press cell.
- Kept plain clicks routed to the TUI so Codex, Gemini, Claude Code, and similar apps remain clickable.

Manual retest confirmed the target CLI/TUI copy bug is fixed.

Remaining follow-up:

- Terminal selection dragging still has noticeable latency and should get a focused performance/refactor pass.
