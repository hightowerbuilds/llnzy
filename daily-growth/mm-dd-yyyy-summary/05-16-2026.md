## Shader reliability

- Reviewed the shader host pipeline from WGPU initialization through WGSL
  pipeline creation, frame encoding, queue submission, staging-buffer readback,
  and CVPixelBuffer conversion.
- Hardened shader initialization and rendering with panic trapping so WGPU
  startup, pipeline creation, or frame rendering failures disable shader
  effects instead of crashing LLNZY.
- Added WGPU validation/internal/out-of-memory error scopes around built-in
  shader pipeline creation and per-frame shader work.
- Added explicit error logs for shader poll/readback failures, CVPixelBuffer
  allocation/lock/unlock failures, render panics, and oversized render-target
  requests.
- Added a render-target size guard to avoid oversized full-window shader
  textures that could trigger Metal/device instability.
- Stopped the shader animation loop after the shader host disables itself, so a
  failed shader path does not keep scheduling dead frames.
- Added a Naga test that parses and validates every built-in WGSL shader and
  confirms each has `vs_main` and `fs_main`.
- Verification:

  - `cargo check --all-targets --all-features`
  - `git diff --check`
