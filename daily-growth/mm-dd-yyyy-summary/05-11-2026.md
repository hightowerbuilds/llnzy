## GPUI Stacker prototype

Added `src/gpui_stacker.rs` and `src/bin/gpui_stacker.rs` behind:

```toml
gpui-stacker = ["dep:gpui"]
```

The prototype uses real Stacker data:

- Loads saved prompts.
- Shows a prompt list.
- Opens a GPUI prompt editor.
- Supports keyboard editing, selection, clipboard, undo/redo, mouse focus, and
  GPUI input-handler integration.

Follow-up in `fdde381`: replaced the initial newline-flattening workaround with
real multiline prompt rendering. GPUI's low-level `shape_line` is single-line,
so the prototype now shapes each prompt line independently and maps cursor,
selection, mouse hit-testing, and IME bounds through that multiline layout.

Known limitation: no wrapping or scrolling yet. This is acceptable for the
prototype stage.

## GPUI editor prototype

Added a separate editor prototype behind:

```toml
gpui-editor = ["dep:gpui"]
```

Files:

- `src/gpui_editor.rs`
- `src/bin/gpui_editor.rs`

The first editor pass is intentionally read-only. It opens a real
`EditorState`, loads an initial file when available, reparses the active
buffer, and renders a GPUI editor surface with line numbers, file metadata,
language label, and status text.

This gives us a low-risk editor beachhead without touching the current editor
runtime or terminal path.
