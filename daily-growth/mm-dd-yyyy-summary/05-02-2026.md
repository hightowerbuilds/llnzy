## Phase 3: State Identity And Session Reliability

Started the state identity work with multiple parallel agents.

Completed:

- Added stable `BufferId` values to `EditorState`.
- Added editor registry helpers for resolving, switching, and closing buffers by `BufferId`.
- Added editor registry helpers for path lookup, path update, and dirty-buffer identity reporting.
- Updated code-file tabs to store `BufferId` instead of editor buffer indexes.
- Updated UI pane metadata and `AppCommand::OpenCodeFile` to carry `BufferId`.
- Moved async parse result application from shifting view indexes to stable buffer IDs.
- Reworked close-tab, modified-tab, save-for-close, and tab-sync paths to resolve buffers by ID.
- Added `AppCommand::RemapCodeFilePath`.
- Remapped open buffers and code-file tabs after sidebar rename.
- Kept file move remapping on the same identity-safe path.
- Blocked `Close Other Tabs` and `Close Tabs To Right` when those actions would remove a dirty code-file tab from the workspace tab bar.
- Repaired last-session restore so startup no longer just deletes the session snapshot.
- Extracted session restore decision-making into a pure restore planner.
- Added backward-compatible `SessionSnapshot.active_tab`.
- Saved and restored active tab when possible.
- Skipped missing restored files safely.
- Fell back to Home when a saved session has no usable tabs.
- Split runtime drag/drop command execution and file path remapping into `src/runtime/drag_drop.rs`, reducing `src/runtime/commands.rs` to 524 lines.
- Split save-prompt response handling into `src/runtime/save_prompt.rs`, bringing `src/runtime/commands.rs` down to 445 lines.
- Added Git UI async request identity guards so stale refresh/detail results cannot overwrite the current repository or selected commit state.
- Added `BufferId` identity guards to buffer-local LSP pending requests and made delayed formatting responses mutate and notify the original request buffer instead of the active buffer at response time.

Added tests for:

- stable buffer IDs after index shifts
- switching by buffer ID after index shifts
- path lookup/update through the buffer registry
- dirty-buffer ID reporting
- async parse still attaching to the correct buffer
- session snapshot round-trip with active tab
- old session snapshots without active tab
- restore planner behavior for normal sessions, missing projects, missing files, active-tab remapping, and Home fallback
- Git UI refresh/detail request identity and stale detail discard behavior
- LSP request identity, delayed formatting after active-buffer switches, and closed-buffer formatting discard behavior
