# 05-13-2026 Summary

## Codebase orientation

Started by re-reading the repository shape after the GPUI migration work. The
important split was between:

- The new GPUI workspace path in `src/gpui_*`.
- Shared product logic that still belongs in the repo: editor, terminal,
  Stacker, sketch, LSP, Git, config, themes, tasks, PTY/session handling, and
  platform paths/packaging.
- The superseded runtime path built around winit, wgpu, egui, the custom
  renderer, and the old app/event-loop modules.

The core product decision was clarified: LLNZY should now be treated solely as
the GPUI workspace app. The old runtime should not remain as a release target,
fallback target, or parallel active app.

## Removed GPUI spike crate

Deleted the old `spikes/gpui_foundation` crate and its build output. That spike
had served its purpose and keeping it in the repository created noise around
which GPUI code was canonical.

Also updated the moved code-quality roadmap/checklist references so they no
longer scan or describe the deleted spike directory.

## GPUI became the default app

Changed the default `llnzy` binary to launch the GPUI workspace directly.

Key changes:

- Replaced the old runtime entrypoint in `src/main.rs` with the GPUI workspace
  entrypoint while preserving startup guardrails:
  - `env_logger`
  - the `llnzy prompt` CLI route
  - panic diagnostics
  - Unix `SIGPIPE` ignore
- Set Cargo defaults to include `gpui-workspace`.
- Updated `README.md` so the build/run instructions describe the GPUI app.
- Updated `bundle.sh` wording so release packaging is explicitly the GPUI
  LLNZY app.

Verified:

The prompt help path exited with usage, as expected for the current CLI shape.

## Dependency prune

Removed direct dependencies that belonged to the old runtime:

- `winit`
- `wgpu`
- `glyphon`
- `egui`
- `egui-wgpu`
- `egui-winit`
- `taffy`
- `pollster`
- direct `bytemuck`
- `arboard`
- direct `objc2`, `objc2-app-kit`, and `objc2-foundation`

Some of those crates still appear in builds as transitive GPUI/macOS graphics
dependencies, but LLNZY no longer depends on them directly or references them
from source.

## Shared API cleanup

Removed winit event-loop proxy APIs from shared code:

- `Pty` no longer takes a `winit::EventLoopProxy`.
- `Session` now uses the proxy-free PTY path.
- `LspNotifier` remains as the GPUI-friendly notifier boundary, while old
  `EventLoopProxy` constructors were removed.
- `UserEvent` was deleted from `src/lib.rs`.

Normalized keybinding parsing away from winit types by introducing local
`Modifiers`, `KeyMatch`, and `NamedKey` types in `src/keybindings.rs`.

Renamed the remaining editor performance field from `egui_ms` to `ui_ms` so the
shared editor model no longer carries stale runtime language.

## Sketch media and Markdown preview

Moved through the Pass 4 sketch-media implementation and the first usable Pass 5
Markdown preview implementation.

Sketch changes:

- Added Sketch Pad image import through the GPUI toolbar.
- Imported images are copied into the app-owned sketch image library.
- Imported images keep their source dimensions when they enter the sketch.
- Imported images now render as actual GPUI image layers instead of placeholder
  boxes.
- Image elements can be selected, moved, resized from the sketch canvas, and scaled
  from selected-image toolbar controls.
- Added a discoverable screenshot drop folder action from the Sketch toolbar.
- Added JPEG export for the whole sketch. When a project is open, exports go to
  a `sketches/` folder inside that project; otherwise the user is asked for an
  export directory.
- JPEG exports preserve the current sketch canvas size instead of applying a width
  cap.

Repo image preview tangent:

- Added read-only previews for common project image files opened from the GPUI
  workspace sidebar, including raster formats and SVG.
- Routed preview image extensions around text-buffer loading so binary image
  files do not enter the editor/LSP/syntax pipeline.
- Added an image preview tab inside the editor surface with dimensions and file
  size metadata.
- Kept editor source-buffer tabs active as normal when selecting text files or
  closing the image preview.
- Guarded editor commands so image previews copy their path and remain read-only.
- Refreshed the local crate build artifacts after a stale incremental linker
  failure, then confirmed the normal GPUI workspace build path links again.

Markdown changes:

- Added source, preview, and split modes for `.md` editor buffers.
- Added the Markdown preview toggle to the editor tab action row.
- Added a separate Markdown page in Appearances so preview styling remains
  distinct from terminal, editor, and sketch settings.
- Implemented a lightweight GPUI Markdown preview renderer for headings,
  paragraphs, bullets, quotes, and fenced code blocks.
