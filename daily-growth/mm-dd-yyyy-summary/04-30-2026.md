## Codebase Orientation

Reviewed the project structure and confirmed `llnzy` is a native Rust desktop app rather than a web app:

- `winit` window/event loop
- `wgpu` rendering
- `glyphon` text rendering
- `egui` overlays and app UI
- `alacritty_terminal` terminal emulation
- `portable-pty` shell sessions
- `tree-sitter` syntax support
- custom LSP management

Key architecture notes:

- `src/main.rs` remains the central app controller and event-loop owner.
- `src/runtime/` contains extracted runtime behavior for commands, layout, tabs, terminal handling, window/session restore.
- `src/ui/` owns egui surfaces such as the footer, tab bar, sidebar, settings, stacker, sketch, and editor/explorer views.
- `src/renderer/` owns GPU rendering, effects, text, cursor, bloom, CRT, particles, and background passes.

## Tab Bar Rename Fix

Fixed a tab rename targeting bug where renamed text could land on a different tab than the user selected. The root cause was that rename detection used a separate manual coordinate calculation based on equal-width tabs, while the actual tab bar renders variable-width egui tabs.

Changes:

- Removed the old manual viewport/equal-width double-click math.
- Moved rename activation to the real per-tab egui `Response`.
- Double-clicking now starts editing the exact tab response that was clicked.
- Inline rename editor now renders inside the actual tab rect.
- `Enter` saves the name to the same tab index.
- `Escape` cancels cleanly.
- Removed stale manual double-click timer state from `UiState`.

Touched:

- `src/ui/tab_bar.rs`
- `src/ui/mod.rs`
