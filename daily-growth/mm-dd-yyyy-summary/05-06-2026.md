## Architecture Understanding

Key takeaways:

- `App` is still the central coordinator and owns the active runtime graph: window, renderer, tabs, UI state, terminal sessions, config, clipboard, and event routing.
- Product surfaces mostly emit typed `AppCommand`s through `UiFrameOutput`, which the runtime layer applies after egui rendering.
- Stable IDs are used where correctness matters: `BufferId` for editor buffers and tab IDs for joined tab groups.
- Async work is usually isolated behind request tokens or stable identities so stale Git/LSP/file-event results do not mutate current state.
- Rendering is in a migration phase: app code still builds `RenderRequest`, but the renderer now adapts it into the app-agnostic `EngineFrame` layer model.
- Security posture is intentionally local-first. Public IPC, extensions, auto-update, enterprise controls, and mutating Git workflows are deferred until stronger permission and governance models exist.
