## Sidebar Layout Retina Fix

Fixed a sidebar layout bug where the visual sidebar partition moved right but
terminal content still rendered underneath it on high-DPI displays.

Root cause:

- egui reports sidebar widths in logical points.
- terminal layout, hit testing, PTY sizing, and renderer geometry use physical
  pixels.
- On Retina displays, the terminal content offset was therefore too small,
  letting the sidebar and bumper cover part of the shell.

Completed:

- Added `logical_to_physical_width` in `src/layout.rs`.
- Added layout tests for normal high-DPI scale factors and invalid scale-factor
  fallback.
- Updated initial bumper layout, sidebar width change detection, terminal
  layout recomputation, cursor hit testing, and native drop targeting to use
  physical sidebar width.
- Kept the egui sidebar itself in logical coordinates while converting only the
  data that crosses into physical window/renderer coordinates.
