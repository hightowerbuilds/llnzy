## Tab Context Menus

Improved the right-click tab menu workflow for joining tabs.

Completed:

- Kept the primary tab context menu anchored directly below the clicked tab.
- Changed `Join Tabs` to open its target menu as a side popup to the right of the primary menu instead of overlapping it.
- Kept outside-click dismissal aware of both popup rectangles so interacting with either menu does not close the menu unexpectedly.
- Removed the extra-click delay so users can right-click a tab, click `Join Tabs` once, and immediately see the side menu.

## Joined Terminal Scrolling

Fixed terminal scrollback routing when terminal tabs are part of a joined layout.

Completed:

- Added joined-terminal pane hit testing for the current pointer position.
- Routed mouse-wheel scrollback to the terminal tab under the cursor instead of always using the active tab.
- Routed mouse-reporting wheel events to the targeted joined terminal tab with pane-relative row/column coordinates.
- Moved terminal-pane wheel routing ahead of egui event handling so joined terminal panes receive scroll input before egui can consume it.
- Preserved trackpad scrolling by converting small non-zero pixel deltas into at least one terminal scroll line.
- Preserved egui-owned scrolling for Markdown and code panes.

This addresses cases where:

- Terminal + Markdown layouts left the terminal unable to scroll when the Markdown pane was active.
- Terminal + CodeFile layouts had the same active-tab routing problem.
- Terminal + Terminal layouts could fail to scroll either shell reliably.

Manual retest confirmed terminal scrolling now works while a shell is in a joined tab.

## Tab Grouping Engine

Started the core tab grouping engine replacement.

Completed:

- Added a dedicated `tab_groups` module with stable tab-id based group state.
- Replaced the UI-level single `Option<JoinedTabs>` state with `TabGroupState`.
- Preserved current two-pane joined rendering while allowing multiple joined groups to exist at once.
- Made group membership independent of tab order, so tab reorder no longer needs index remapping for groups.
- Made close/retain behavior dissolve only groups whose stable tab ids are no longer open.
- Kept group-local active tab and divider ratio in the group model.
- Added `Swap Tabs` to joined-tab context menus so users can flip the left and right panes.
- Added separate inline rename controls for joined tabs: one button edits the left tab name and one edits the right tab name.
- Added focused tests for multiple groups, separating one group without affecting another, retaining groups after close, and rendering multiple joined tab-bar entries.

Manual retest confirmed multiple grouped terminal panes can run TUIs and switch between them.
