# 05-09-2026 Summary

## Jaded Review Read-Through

Started the session by reading `daily-growth/roadmaps/jaded-review.md` and
focusing on its sharpest criticism: that the Stacker editor uses a WebView
inside an otherwise wgpu/glyphon-rendered app, undermining the native-austerity
story.

## Why The WebView Existed

Traced the actual reason through git log and the May 4 build summary:

- The WebView landed in commit `7af1efd` (`Add WebView-backed Stacker editor`).
- The summary on `05-04-2026.md` records the trigger plainly: voice dictation,
  specifically Wispr Flow. macOS dictation, IME, and AutoFill all hook the OS
  text-input protocol (`NSTextInputClient`). egui's painted `TextEdit` does
  not satisfy that protocol.
- A first attempt used a hidden invisible `NSTextView` bridge in
  `src/macos_text_bridge.rs`. It was fragile because dictation needs real
  on-screen geometry to anchor its overlay, and focus regains had races.
- The WebView was a pragmatic workaround. WKWebView is already a fully featured
  `NSTextInputClient` with on-screen geometry, so dictation and Wispr Flow
  delivered cleanly.

## Roadmap For Native Input

Recorded the path forward in
`daily-growth/roadmaps/stacker-native-input-roadmap-05-09-2026.md`:

- **Option A (now):** Replace the WKWebView with a visible `NSTextView` inside
  an `NSScrollView` overlay. Same architectural shape (native subview clipped
  to the Stacker pane), no JS/HTML/Chromium, dictation still routed by AppKit.
- **Option C (follow-up):** Phased plan to fold Stacker into the existing
  editor as a `BufferKind::Prose` buffer rendered through glyphon, with
  `NSTextInputClient` implemented on the winit content view itself.

## Option A Implementation

Completed the cutover end-to-end.

Added:

- `src/stacker_native_view.rs` — visible `NSTextView` inside `NSScrollView`,
  subclassed via `define_class!` to capture `didChangeText` and
  `didChangeSelection`.
- A typed `UserEvent::StackerNativeTextChanged { kind, text, utf16_start,
  utf16_end }` variant replacing the old JSON-payload
  `UserEvent::StackerWebViewMessage(String)` and the synthesized
  `UserEvent::StackerNativeEdit(StackerNativeEdit)`.
- Visual styling matched to the prior WebView textarea: dark background, soft
  white text, green caret, 34pt text-container insets, 16pt system font with
  configurable size.

Changed:

- `lib.rs` — replaced `stacker_webview` with mac-only `stacker_native_view`.
  Removed the `StackerNativeEdit` struct.
- `main.rs` — renamed `stacker_webview` → `stacker_native_view`,
  `stacker_webview_pending_focus` → `stacker_native_view_pending_focus`, and
  the `create_*`, `sync_*`, and `apply_*` methods accordingly. Deleted
  `sync_macos_text_bridge` and the `stacker_bridge_active` /
  `stacker_bridge_text` fields; the visible NSTextView is its own first
  responder, so the hidden bridge is redundant.
- `main_app/lifecycle.rs` — dropped `macos_text_bridge::setup` and `install`
  calls; renamed the create call.
- `main_app/frame.rs` — dropped the per-frame `sync_macos_text_bridge` call;
  renamed the native-view sync call.
- `main_app/events.rs` — replaced the JSON-decoding webview-message branch and
  the synthesized native-edit branch with a single typed
  `StackerNativeTextChanged` branch.
- `main_app/handler.rs` — removed the focus-regain bridge resync.
- `runtime/terminal.rs` — removed `apply_stacker_native_edit` and the
  `StackerNativeEdit` import; the visible view sends typed text-changed
  events directly so synthesized-edit reconstruction is no longer needed.
- `runtime/tabs.rs`, `runtime/external_commands.rs` — pending-focus field
  rename, mac-gated.
- `Cargo.toml` — removed `wry` and the standalone `dpi` crate. Added
  `NSColor`, `NSFont`, `NSScrollView` features to `objc2-app-kit`.
- `docs/stacker-command-workflow-05-05-2026.md` — updated the document model
  contract to reflect the native overlay; clarified that the prior WebView
  and hidden AppKit bridge have been removed.

Deleted:

- `src/stacker_webview.rs`
- `src/macos_text_bridge.rs`
