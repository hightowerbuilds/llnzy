# llnzy — Build Session Summary
## April 14–15, 2026

---

## What We Built

A fully functional, GPU-accelerated terminal emulator from scratch in Rust. Started from an empty directory and implemented all 13 phases across two sessions — from first window to tested, documented, packaged macOS app.

**Final codebase:** ~5,500+ lines of Rust across 14 source files, plus 234 tests and full CI/CD.

## Phases Completed

### Phase 1: Foundation
- Cargo project scaffold
- winit window with wgpu GPU surface
- glyphon text rendering pipeline
- PTY spawning via portable-pty
- alacritty_terminal integration (ANSI parsing, grid state)
- Terminal grid to GPU rendering pipeline
- Keyboard input encoding to escape sequences
- Window resize propagation to PTY + grid + surface
- Bundled JetBrains Mono font (embedded via include_bytes)

### Phase 2: Core UX
- Cursor rendering (Block, Beam, Underline styles)
- Scrollback with mouse wheel and Shift+PageUp/Down
- Cell background colors (batched non-default bg rects)
- Text selection via click-and-drag with highlight rendering
- Clipboard: Cmd+C copy, Cmd+V paste, Cmd+A select all
- Accurate cell sizing using actual glyph metrics from layout_runs
- TOML config file loading from ~/.config/llnzy/config.toml
- 256-color palette + true color (RGB) support
- Auto-scroll to bottom on keyboard input

### Phase 3: Text Rendering Quality
- Bold attribute rendering using bundled Bold font
- Italic rendering with bundled Italic + BoldItalic fonts
- Underline decorations: single, double, curly (undercurl), dotted, dashed
- Strikethrough decoration
- Configurable font family via config
- Font fallback chain (FontSystem::new loads system fonts for emoji, CJK, symbols)
- DPI/HiDPI/Retina scaling awareness (font size * scale_factor)
- Ligature support with toggle (Shaping::Advanced vs Basic)
- INVERSE flag (swap fg/bg), DIM (33% brightness reduction), HIDDEN

### Phase 4: Input & Interaction
- Alt/Option key combinations (ESC prefix)
- Full modifier matrix: Ctrl+Shift, Ctrl+Alt with CSI modifier parameters
- Application cursor mode (DECCKM: \x1bOA vs \x1b[A)
- Double-click word selection (500ms window)
- Triple-click line selection
- Mouse reporting protocol (SGR and X10 encoding for vim, htop, etc.)
- Bracketed paste mode (\x1b[200~ ... \x1b[201~)
- URL detection + Cmd+click to open (text scan + OSC 8 hyperlinks)
- Right-click: copy if selection exists, paste if not

### Phase 5: Terminal Compliance
- Channel-based EventProxy replacing no-op listener
- Window title from OSC 0/2 escape sequences
- OSC 52 clipboard integration
- OSC 8 hyperlink support (cell-level URI access)
- PtyWrite event forwarding
- Visual bell (150ms white flash overlay on BEL character)
- Terminal mode flag queries: APP_CURSOR, BRACKETED_PASTE, MOUSE_MODE, SGR_MOUSE, ALT_SCREEN
- Child exit detection with exit code tracking
- TERM=xterm-256color environment

### Phase 6: Performance
- Line-level dirty tracking via content hashing (DefaultHasher per row)
- Text buffer caching: Vec<Buffer> persists across frames, only dirty lines reshaped
- Pre-allocated reusable vertex buffer for rects (grows with next_power_of_two)
- Adaptive frame rate: ControlFlow::Wait, redraws only on content change
- PTY output batching: 64KB read buffer, all chunks concatenated before processing
- GPU atlas trimming after each frame
- Cache invalidation on resize, scroll, tab switch

### Phase 7: Configuration & Theming
- Full 16 ANSI color scheme configurable in TOML
- 5 built-in color scheme presets: Dracula, Nord, One Dark, Solarized Dark, Monokai
- Window padding (padding_x, padding_y)
- Window opacity/transparency (with_transparent + alpha in clear color)
- Configurable line height multiplier
- Font weight and style configuration
- Cursor blink with configurable rate (ms), resets on keypress
- Selection color and alpha customization
- Scroll speed (lines per wheel tick)
- Config hot-reload: mtime check every 2 seconds, applies without restart

### Phase 8: Multi-Session & Layout
- Session module: Terminal + PTY + title as a self-contained unit
- Tab support: Vec<Tab> with active_tab index
- Tab bar: rendered at top when 2+ tabs, shows titles from OSC sequences
- Cmd+T new tab, Cmd+W close tab, Cmd+1-9 switch, Cmd+]/[ next/prev
- PaneNode tree: Leaf(Session) | Split { dir, ratio, first, second }
- Vertical split (Cmd+D) and horizontal split (Cmd+Shift+D)
- Pane divider rendering (1px lines)
- Pane focus cycling (Cmd+Arrow)
- All PTY outputs processed for all tabs (background tabs don't stall)
- Pane resize_all propagates to all sessions in the tree

### Phase 9: Search & Navigation
- Cmd+F opens search bar at bottom of window
- Incremental case-insensitive substring search (updates as you type)
- All matches highlighted in amber, focused match in brighter amber
- Enter = next match, Shift+Enter = previous match
- Status display: "3/15" or "No matches"
- Ctrl+R toggles regex mode (full regex crate support)
- Escape closes search and clears highlights
- Searches all visible rows in the current viewport

### Phase 10: System Integration
- Shell exit detection: tabs auto-close when shell exits, respawns if all exit
- Window titlebar updates from active session title: "{title} — llnzy"
- Cmd+Enter toggles borderless fullscreen
- Drag-and-drop files: inserts shell-escaped path into terminal
- Window state persistence: size saved to ~/.config/llnzy/window_state.toml on close, restored on launch
- OSC 52 clipboard store events forwarded to system clipboard

### Phase 11: Error Handling & Stability
- Built-in diagnostics panel (Cmd+Shift+E): 40% height overlay showing all logged events
- Thread-safe ErrorLog with ring buffer (1000 entries), three levels (Info/Warn/Error)
- Timestamped entries with color coding: blue info, amber warn, red error
- Panel scrollable with Arrow/PageUp/PageDown, closeable with Escape
- Header shows total counts: "Diagnostics — 47 entries (2 errors, 5 warnings)"
- Custom panic handler logging to stderr
- All error paths pipe to the diagnostics panel
- Config reload, tab failures, split failures, shell exits all logged
- Graceful shutdown with window state persistence

### Phase 12: Testing & Quality
- **175 unit tests** across 7 modules: config (hex parsing, indexed color, presets, TOML apply), input (arrow keys, tilde keys, function keys, mouse encoding, modifier-aware named keys), selection (range normalization, rects, lifecycle), search (plain/regex matching, navigation, status), error_log (counts, recent, cap, thread safety, panel rendering), terminal (creation, resize, cell access, ANSI processing, color resolution, mode flags, events), session (split_rect geometry)
- **45 integration tests**: cursor movement, erase operations, line wrapping, scroll regions, text attributes (bold, italic, underline, dim, hidden, inverse, strikethrough), 16/256/true color, OSC sequences, terminal modes, alternate screen buffer, tab stops, character insert/delete, resize, search + selection on processed terminal
- **5 PTY round-trip tests**: shell spawn + prompt, echo command, resize, multi-command, reader close on shell exit
- **CI/CD pipeline**: GitHub Actions with check, fmt, clippy (-D warnings), unit tests, integration tests, PTY tests, release build with artifact upload
- Refactored to lib + bin crate structure for testability
- Zero clippy warnings, rustfmt clean
- Added Debug/Default derives where needed
- Boxed Session in PaneNode::Leaf to fix large enum variant

### Phase 13: Documentation & Distribution
- **README.md**: Complete rewrite with honest feature descriptions, "What it doesn't do (yet)" section, build/test/lint instructions, config quickstart, keyboard shortcuts by category, project structure, test coverage summary
- **docs/configuration.md**: Full config reference — every TOML key with type, default, description. All 5 presets, all 16 ANSI overrides, hot-reload behavior documented
- **LICENSE**: MIT
- **CHANGELOG.md**: Keep a Changelog format, full unreleased entry
- **macOS .app bundle**: Info.plist with HiDPI support, proper bundle structure so it launches as a native GUI app without Terminal.app

### Input System Overhaul (post-Phase 13)
- Fixed space key bug: winit 0.30 sends Space as NamedKey::Space, was silently dropped
- Fixed Ctrl+Space: now correctly sends NUL (0x00) instead of plain space
- Fixed Alt+Space: now sends ESC + space for proper terminal Alt behavior
- Fixed Alt+Backspace: now sends ESC + DEL for word-delete in shells
- Fixed Cmd key leaking: unrecognized Cmd combos (Cmd+Z, etc.) no longer send stray characters to the terminal
- Fixed named key fallthrough: unhandled Named keys now check event.text instead of returning None
- Refactored ctrl_code() as dedicated function handling both Named and Character keys
- Added modifier awareness to all named key encoding (Space, Backspace respect Ctrl/Alt)
- Added IME support: WindowEvent::Ime(Commit) feeds composed text to terminal or search bar — enables non-US keyboards, dead keys, CJK input methods
- Increased default padding from 2px to 8px
