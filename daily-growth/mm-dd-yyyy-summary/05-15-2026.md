## Shader / wgpu crash hardening

- Triggered investigation: long session with shaders + terminal output
  crashed via `wgpu error: Out of Memory` panic. Crash log + persisted
  error log captured the exact cause — diagnostic pipeline validated
  end-to-end on a real failure.
- **Pooled wgpu render texture + staging buffer** in `EffectsHost`
  keyed by `(width, height)`. Steady-state shader rendering now does
  zero per-frame GPU allocation; resizes invalidate and rebuild.
  Defensive `mapped` flag prevents the pooled staging buffer from being
  left mapped on early-return paths.
- **Custom wgpu uncaptured-error handler** installed via
  `device.on_uncaptured_error(...)`. On OOM / validation / internal
  error: logs via `log::error!` (lands in error log), sets a disabled
  flag, future `render_frame` calls return `None`. App no longer crashes
  on GPU memory pressure — shader silently pauses, editor stays alive.
- **Replaced live shader preview in Appearances** with a static
  three-band palette swatch. One fewer concurrent shader surface; the
  real terminal background still runs the live shader.

## Markdown handling in the code editor

- `.md` / `.markdown` / `.mdown` / `.mkd` files now default to **Preview**
  on open. The screenshot showed raw markdown source rendered with no
  syntax awareness; opening in Preview avoids that entirely.
- Inline **Preview / Source** toggle pill rendered as an absolute overlay
  at the top-right of the editor body. No layout space consumed, so the
  body keeps full height in both modes.
- Blog-style preview typography: Atkinson Hyperlegible, 16px / 26px line
  height, comfortable 720-820px max width, generous block gap, larger
  heading hierarchy (32 / 24 / 19 / 16 px), `pb_24` bottom padding so
  scroll bottoms out with breathing room. Left-aligned with `pl_8`
  padding (was centered, user preferred left).
- Source mode: rulers (`appearance.rulers`, default `[80, 120]`) skipped
  for markdown files — the "weird vertical stripe down the right side"
  was the column-80 ruler line drawing through empty space on short
  prose lines.
- Scroll/sizing fixes: the preview wouldn't scroll because
  `markdown_preview_body`'s `flex_1` was being silently ignored by taffy
  when its parent wasn't a flex container. Settled on a relative-wrapper
  + absolute size_full scroll-child pattern so the scroll has a defined
  bounded height regardless of flex chain quirks.
- Source mode "half blocked" diagnosis: traced to flex sizing combined
  with short markdown line content. The `min_h(px(0.0))` + explicit
  `.flex().flex_col()` parent fixed cases where flex children weren't
  shrinking below their natural height.
- Required killing + relaunching the running app each time, not just
  rebuilding — `open target/llnzy.app` focuses an existing instance
  instead of launching the new bundle. Confirmed by binary timestamps
  and pid checks.

## Stacker — prompt management

- **Blank startup**: editor opens empty, `active_prompt: None`. No more
  pre-loading the first saved prompt. The saved-prompt sidebar list is
  the only entry point for editing existing prompts.
- **Save button** (green-tinted, toolbar right): updates active prompt in
  place, or creates a new prompt from the current text. Calls
  `persist_prompt_library` for atomic `.md` writes.
- **+ New button**: resets the editor to a blank draft and clears
  `active_prompt` so the next save creates rather than updates.
- **Delete per prompt** (× button on each row): opens a centered
  confirmation modal with Cancel / Delete buttons + a dim scrim that
  also cancels. Confirmed delete removes the prompt, persists the
  library, and resets the editor if the deleted prompt was active.
  Modal click is `stop_propagation`'d so clicks inside the card don't
  cancel.
- **Lightweight autosave**: 1.5-second background tick captures the
  editor text snapshot. Saves only when:
  (a) the text matches the previous tick's snapshot (user paused for
  ≥1.5s), AND (b) the text differs from what's stored on disk for the
  active prompt. Works for both updating active prompts and
  auto-creating new prompts from a non-empty draft after the pause.
  Loading a prompt and the manual Save button both sync the autosave
  baseline so they don't double-write.
- New constructor params on `StackerPrototype`: `pending_delete:
  Option<usize>`, `autosave_previous_text: String`.

## Stacker — autosave switched from timer to event-driven

- Killed the 1.5-second `AUTOSAVE_TICK` background loop and the
  `autosave_if_idle` method + `autosave_previous_text` field. No more
  wakeups while idle.
- **Tab close hook**: in `WorkspacePrototype::close_tab`, when the
  closing tab has `WorkspaceSurface::Stacker`, save the active prompt
  before removing the tab. The Stacker entity stays alive (workspace
  singleton) but the user's mental model is "closing the tab = closing
  the work."
- **App quit hook**: `run_workspace_prototype` now captures the window
  handle (it's `Copy`) and the `Quit` action handler calls a new
  `WorkspacePrototype::save_drafts_before_quit(cx)` method before
  `cx.quit()`. Save errors are logged but never block the quit.
- **Coverage caveat**: hard kill / SIGKILL / OS-initiated shutdown
  without graceful quit won't run the handler. For Cmd+Q, menu Quit,
  and normal close, work persists.

## Stacker — fixed editor collapse after divider change

- Initial divider change wrapped `editor_panel(...)` in an extra
  `div().flex_1().child(...)`. The wrapper wasn't a flex container, so
  the panel's internal `flex_1` chain broke and the editor body
  collapsed to content height (~0). Result: a large empty dark area
  where the editor should be, no text visible.
- Fix: removed the extra wrapper. `editor_panel` already returns a div
  with its own `flex_1 + min_h(320)` so it fills the remaining vertical
  space directly as a flex_col child of the workbench. The dark area
  was the CONTENT_BG panel background showing through with no editor
  entity painted inside.
