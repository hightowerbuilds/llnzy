## Terminal — open in project root

- `TerminalSurface::new_with_cwd(cwd, cx)` threads cwd through to
  `launch_session`. Initial terminal uses `std::env::current_dir()`;
  every new terminal opened after `open_project` uses the new
  workspace root. Existing terminals keep their original cwd
  (matches per-shell mental model).

## Shader / wgpu crash hardening

- Triggered investigation: long session with shaders + terminal output
  crashed via `wgpu error: Out of Memory` panic. Crash log + persisted
  error log captured the exact cause — diagnostic pipeline validated
  end-to-end on a real failure.
- **Pooled wgpu render texture + staging buffer** in `EffectsHost`
  keyed by `(width, height)`. Steady-state shader rendering now does
  zero per-frame GPU allocation; resizes invalidate and rebuild.
  Defensive `mapped` flag prevents the pooled staging buffer from being
  left mapped on early-return paths.
- **Custom wgpu uncaptured-error handler** installed via
  `device.on_uncaptured_error(...)`. On OOM / validation / internal
  error: logs via `log::error!` (lands in error log), sets a disabled
  flag, future `render_frame` calls return `None`. App no longer crashes
  on GPU memory pressure — shader silently pauses, editor stays alive.
- **Replaced live shader preview in Appearances** with a static
  three-band palette swatch. One fewer concurrent shader surface; the
  real terminal background still runs the live shader.
