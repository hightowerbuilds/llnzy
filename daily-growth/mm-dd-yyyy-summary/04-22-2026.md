# llnzy — Build Session Summary
## April 22, 2026

---

## What We Built

Transformed llnzy from a functional terminal emulator into a visually dynamic, video-game-like experience. Added a full GPU effects pipeline — animated shader backgrounds, post-processing bloom/glow, CRT retro effects, a compute-shader particle system, glowing cursor with trails, and text entrance animations — all running at 60fps on Metal.

**New files:** 7 renderer modules (~1,800 lines), plus config and infrastructure changes across 6 existing files.

---

## Completed Phases

### Phase 0: Rendering Infrastructure
- Added `FrameUniforms` (time, delta_time, resolution, frame count) shared across all shaders
- Created offscreen scene textures (A + B for ping-pong post-processing)
- Built `BlitPipeline` with fullscreen triangle WGSL shader
- Added `[effects]` config section with hot-reload support
- Continuous animation mode (`ControlFlow::Poll`) when effects enabled, VSync-paced
- Conditional rendering: effects off = direct-to-swapchain (zero overhead)

### Phase 1: Animated Background Shader
- Built `BackgroundRenderer` with domain-warped fractal Brownian motion smoke shader
- Double-warped noise (4 octaves per warp, 5 octaves final) for organic flowing motion
- Blue-grey color palette tuned from terminal background color
- Configurable intensity, speed, and effect type via TOML
- Also prototyped a snowy mountains + fog shader (reverted to pure smoke by preference)

### Phase 2: Post-Processing Pipeline
- **Bloom/Glow**: 4-pass pipeline (threshold extraction, H-blur, V-blur, additive composite) at half-resolution with 9-tap Gaussian kernel
- **CRT/Retro**: single-pass shader with scanlines, barrel distortion, vignette, chromatic aberration, and phosphor-style effects
- **Film grain**: hash-based animated noise overlay
- Ping-pong scene textures for chaining bloom -> CRT without read/write conflicts
- All effects independently togglable and hot-reloadable

### Phase 3: GPU Particle System
- Compute shader (`@workgroup_size(256)`) updating 1,500+ particles per frame entirely on GPU
- Storage buffer for particle state (position, velocity, color, life, size)
- Pseudo-random respawn with hash-based seeding, sine wobble drift, edge wrapping
- Instanced quad rendering with soft circular falloff and additive blending
- Pulsing brightness and life-based alpha fade-in/fade-out

### Phase 4: Cursor Effects + Text Animations
- **Cursor glow**: SDF-based radial falloff shader with soft halo around cursor
- **Cursor pulse**: brightness oscillation via `sin(time)`
- **Cursor trail**: ring buffer of last 12 positions rendered as fading ghost instances with quadratic alpha decay
- **Text entrance**: lines fade in (alpha 0->1) and slide up from below over ~200ms with smoothstep easing
- Line age tracking per visible row, resets when content hash changes

## Bugs Fixed

- **Config path**: macOS uses `~/Library/Application Support/llnzy/config.toml`, not `~/.config/`
- **Grey screen**: Scene texture blit always active — added conditional bypass when effects off
- **`VERTEX_WRITABLE_STORAGE` crash**: Split compute (read-write) and render (read-only) particle bind groups
- **WGSL array indexing**: `let` arrays can't be indexed by runtime vars — replaced with `switch`
- **Choppy animation**: Manual `WaitUntil` timing — switched to `ControlFlow::Poll` with VSync
- **Scene texture conflict**: Can't read + write same texture in one pass — added second scene texture for ping-pong
- **Cursor shader visibility**: Trail buffer needed `VERTEX_FRAGMENT` visibility, not just `VERTEX`
