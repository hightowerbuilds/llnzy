# llnzy — Build Session Summary
## April 22, 2026

---

## What We Built

Transformed llnzy from a functional terminal emulator into a visually dynamic, video-game-like experience with a proper UI framework. Added a full GPU effects pipeline (smoke, bloom, particles, CRT, cursor glow, text animations), integrated egui for UI chrome, added a footer nav bar with view navigation, and built a settings panel — all running at 60fps on Metal.

---

## Session 1: GPU Visual Effects Pipeline (Phases 0-4)

### Phase 0: Rendering Infrastructure
- `FrameUniforms` (time, delta_time, resolution, frame count) shared across all shaders
- Dual offscreen scene textures for ping-pong post-processing
- `BlitPipeline` with fullscreen triangle WGSL shader
- `[effects]` config section with hot-reload
- Continuous animation mode (`ControlFlow::Poll`) with VSync pacing
- Conditional rendering: effects off = direct-to-swapchain (zero overhead)

### Phase 1: Animated Background Shader
- `BackgroundRenderer` with domain-warped fractal Brownian motion smoke
- Double-warped noise (4 octaves per warp, 5 octaves final)
- Subtle blue-grey palette derived from terminal background color
- Also prototyped snowy mountains + fog shader (reverted to pure smoke)

### Phase 2: Post-Processing Pipeline
- **Bloom/Glow**: 4-pass (threshold, H-blur, V-blur, additive composite) at half-res
- **CRT/Retro**: scanlines, barrel distortion, vignette, chromatic aberration
- **Film grain**: hash-based animated noise overlay
- All effects independently togglable and hot-reloadable

### Phase 3: GPU Particle System
- Compute shader (`@workgroup_size(256)`) updating 800+ particles per frame
- Pseudo-random respawn, sine wobble drift, edge wrapping
- Instanced quad rendering with soft circular falloff and additive blending

### Phase 4: Cursor Effects + Text Animations
- SDF-based cursor glow with radial falloff + pulse + 12-position trail
- Text entrance: lines fade in and slide up over ~200ms with smoothstep easing

## Session 2: UI Framework + Navigation

### Taffy Layout Engine Integration
- Added `taffy` for CSS flexbox layout computation
- `ScreenLayout` struct computes all zones (tab bar, content, footer) from one source
- Replaced ad-hoc pixel math for content positioning
- `pixel_to_grid()` and `grid_to_pixel()` use layout zones

### egui Integration
- Added `egui`, `egui-wgpu`, `egui-winit` for UI chrome rendering
- egui handles footer nav bar, settings panel, button rendering, text layout, DPI scaling
- Custom shaders still handle terminal content + all visual effects
- Resolved wgpu 22 `RenderPass<'static>` lifetime issues with `forget_lifetime()`
- egui renders AFTER terminal pipeline to layer on top

### Footer Navigation Bar
- Persistent footer with three buttons: **Shells**, **Stacker**, **Settings**
- Active view highlighted with brighter button background
- Shells = terminal view, Settings = settings panel, Stacker = placeholder
- Left-aligned buttons with 8px padding, 16px font

### Settings Panel
- Full-screen panel rendered by egui when Settings is active
- Sidebar with Background and Text tabs (selectable labels)
- Grid layout showing current config values
- Navigation via footer bar (no separate Back button)

### Layout & Rendering Fixes
- Fixed first character clipping ("L" in prompt) via `glyph_offset_x`
- Fixed cursor sitting above text line via `glyph_offset_y` baseline alignment
- Fixed smoke being too bright/opaque (darkened palette, reduced intensity)
- Fixed footer not visible (egui render order — must render after terminal submit)
- Fixed config path (macOS: `~/Library/Application Support/llnzy/config.toml`)
