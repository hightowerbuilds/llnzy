# llnzy — Build Session Summary
## April 22, 2026

---

## What We Built

Transformed llnzy from a functional terminal emulator into a visually dynamic, video-game-like experience with a full UI framework. Added GPU visual effects (smoke, bloom, particles, CRT, cursor glow, text animations), integrated egui for UI chrome, built a footer nav bar with three-view navigation (Shells/Stacker/Settings), an interactive settings panel with real-time config changes, and a Stacker prompt queue manager.

---

## Session 1: GPU Visual Effects Pipeline (Phases 0-4)

### Phase 0: Rendering Infrastructure
- `FrameUniforms` shared across all shaders, dual offscreen scene textures
- `BlitPipeline`, continuous animation mode, conditional rendering

### Phase 1: Animated Background Shader
- Domain-warped fbm smoke with subtle blue-grey palette

### Phase 2: Post-Processing Pipeline
- Bloom/Glow: 6-pass (threshold, 2x H+V blur, composite) with 13-tap Gaussian, wide smoothstep threshold, brightness clamping
- CRT/Retro: scanlines, barrel distortion, vignette, chromatic aberration, film grain
- All effects as collapsible sections in settings

### Phase 3: GPU Particle System
- Compute shader particles with instanced quad rendering and additive blending

### Phase 4: Cursor Effects + Text Animations
- SDF cursor glow with pulse + 12-position trail
- Text entrance fade+slide animations

---

## Session 2: UI Framework + Navigation

### egui Integration
- Replaced hand-built UI chrome with egui (0.29) + egui-wgpu + egui-winit
- Resolved wgpu 22 `RenderPass<'static>` lifetime issues with `forget_lifetime()`
- egui renders AFTER terminal pipeline via separate command encoder submission

### Taffy Layout Engine
- CSS flexbox layout for screen zones (tab bar, content, footer)
- `ScreenLayout` as single source of truth for geometry

### Three-View Navigation
- `ActiveView` enum: Shells, Stacker, Settings
- Footer nav bar always visible with highlighted active view
- Views switch instantly via footer buttons

### Interactive Settings Panel
- Background: type dropdown (smoke/none), intensity/speed sliders — changes apply in real-time
- Bloom/Glow: collapsible section with enabled toggle, threshold/intensity/radius sliders
- Particles: collapsible section with enabled toggle, count/speed sliders  
- CRT/Retro: collapsible section with enabled toggle, scanlines/curvature/vignette/chromatic aberration/grain sliders
- Cursor & Animation: text animation, cursor glow, cursor trail toggles, cursor style dropdown, blink rate slider
- Config changes flow: egui → pending_config → renderer.update_config() on same frame
- File hot-reload paused for 10 seconds after UI changes to prevent overwriting

### Stacker — Prompt Queue Manager
- Full-screen view for managing reusable prompts
- Multiline text input with "Save to Queue" button
- Auto-labels from first 6 words of prompt text
- Scrollable queue list with prompt preview
- Copy to clipboard button (via arboard)
- Delete button to remove prompts

---

## Key Fixes

- Bloom flicker: widened threshold smoothstep, added brightness clamping, doubled blur passes, 13-tap kernel
- Cursor drift: reverted cell_width to exact glyph advance (ceil caused cumulative drift)
- Cursor baseline: added glyph_offset_y to align cursor with text
- First character clipping: fixed with 4px glyph_offset_x padding
- Config not applying: file hot-reload was overwriting UI changes
- All effects default to OFF — users enable what they want
- egui render order: must render after terminal encoder submit
