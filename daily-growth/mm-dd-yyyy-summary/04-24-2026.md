# llnzy — Build Session Summary
## April 24, 2026

---

## What We Built

Added a toggleable navigation sidebar, integrated the UI into the GPU shader pipeline, themed the chrome to match visual presets, and restructured the view navigation with a new Appearances/Settings split.

---

## Changes Made

### Navigation Sidebar (Cmd+B)
- New left sidebar panel (200px) toggled with Cmd+B
- Shared across all shell tabs — same sidebar regardless of active tab
- Contains navigation buttons: Shells, Stacker, Appearances, Settings
- Terminal content area and tab bar shift right and resize when sidebar opens
- PTY sessions re-fit to new grid dimensions on toggle
- `ToggleSidebar` action added to keybinding registry

### Sidebar Theming
- Sidebar and footer colors derived from active theme's color scheme
- Background: 65% of terminal bg color; active button: 40% of cursor color
- Text uses theme foreground color
- Colors update live when switching themes

### GPU Shaders Affect the Sidebar
- Rendering pipeline restructured: egui now renders to the offscreen scene texture before post-processing (when on Shells view)
- Bloom, CRT scanlines, vignette, chromatic aberration, and film grain all apply to the sidebar
- Settings/Stacker/Appearances views render clean (egui goes to swapchain after post-processing)
- `apply_effects_to_ui` flag on `RenderRequest` controls the routing

### Theme & Background Updates
- Removed Cyberpunk theme
- Renamed Retro theme to "Buzz" — now includes green-tinted smoke background
- Removed matrix, tron, nebula from background dropdown (kept smoke, aurora)
- Added custom background color picker: checkbox to enable, egui color picker to choose hue for smoke/aurora
- `background_color: Option<[u8; 3]>` field added to `EffectsConfig`

### View Restructure
- Old "Settings" view renamed to "Appearances" (themes, backgrounds, text/cursor)
- New blank "Settings" view added for future use
- `ActiveView` enum now has four variants: Shells, Stacker, Appearances, Settings
