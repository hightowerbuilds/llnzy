# 06-07-2026 Refactor Summary

## Breadcrumbs

Top-down scan of the June 7 refactor session.

**Code review follow-through**
- Converted the code-review findings into a single refactor pass focused on
  workspace lifecycle safety, Stacker persistence, Light Mode durability,
  Settings palette consistency, and clippy cleanup.
- Kept the edits scoped to the reviewed surfaces instead of expanding into
  unrelated UI or architecture changes.

**Workspace lifecycle safety**
- Replaced the workspace Cmd+Q handler's captured-window save path with a
  registry of live workspace entities.
- Quit now gives each live workspace a chance to save Stacker drafts and the
  workspace recovery snapshot before `cx.quit()`.
- Changed workspace recovery persistence to return errors so quit-time failures
  can be logged as real save failures.
- Added a project open/close guard that refuses to switch projects while any
  open editor buffer is modified.
- The project guard reports the first dirty editor buffer through the sidebar
  status instead of force-closing file editor tabs.

**Stacker persistence**
- Changed `persist_prompt_library` and `persist_prompt_library_at` to return
  `Result<(), String>` instead of only logging disk failures.
- Threaded Stacker save/delete failures into the GPUI Stacker surface.
- Added footer status messages for successful prompt saves, prompt deletes,
  and failed persistence.
- Updated the Stacker Save button, tab-close save path, and quit-save path to
  handle returned persistence errors.
- Added delete rollback on failed archive/write so the in-memory prompt list
  stays aligned with disk state.

**App theme persistence**
- Added an `app_theme` workspace preference for the selected built-in app-wide
  theme.
- Startup now restores the saved app theme before applying terminal-specific
  background image, image fit, palette, intensity, font, layout, editor syntax,
  and word-wrap preferences.
- Added `preserve_terminal_effects` to visual themes so Light Mode can restore
  app-wide light colors without disconnecting the user's terminal background
  effect setup.
- User-loaded themes default to full effect replacement, while the built-in
  Light Mode preserves terminal effects.

**Light Mode Settings polish**
- Passed `WorkspacePalette` through shared Settings controls that were still
  using dark constants.
- Updated the Settings Editor word-wrap row, Advanced Error Log filters,
  Error Log actions, Error Log rows, empty states, and clear-confirm modal to
  render from the active palette.

**Clippy cleanup**
- Fixed editor input clippy warnings for manual modulo checks and redundant
  comparisons.
- Reduced Sketch dashed-boundary helper argument count by introducing a small
  `DashedLineSpec`.
- Added narrow `#[expect(clippy::too_many_arguments)]` annotations for large
  GPUI render helpers whose wide signatures intentionally pass explicit
  view-model fields.

## Verification

- `cargo check --all-targets --all-features`
- `cargo clippy --all-targets --all-features -- -D warnings`
- `cargo test --all-targets --all-features`
- `cargo test --release --test performance_budgets -- --ignored --nocapture`
- `cargo fmt --check`
- `git diff --check`

All verification passed after the final delete-rollback adjustment.

# 06-07-2026 Code Review & Cleanup Pass

## Breadcrumbs

A full code-review pass aimed at removing stale/dead/suppressed code and
modularizing oversized files. Findings were gathered with parallel audit
agents and every dead-code claim was cross-referenced (whole-`src/` reference
counts) before deletion, since this is a lib crate where the compiler does not
flag unused `pub` items.

**Audit baseline (what was already clean)**
- ~59,600 LOC compiled warning-free with no unused dependencies.
- No TODO/FIXME/HACK markers, no commented-out code, no `todo!`/`unimplemented!`,
  no `#[ignore]`d tests, no orphaned `.rs` files.
- Only 4 `#[allow(...)]` attributes existed, all justified (LSP `root_uri`
  deprecation compatibility shims and one clippy `mutable_key_type` on a test).
- One stale doc left in place: the `effects.rs` "M1 status" header still
  describes work that shipped in M2 (noted, not changed).

**Dead code removal (~1,300 LOC, verified zero-reference)**
- Deleted two fully-unreachable modules: `external_command` and
  `editor/project_search` (and their `pub mod` declarations + a stale doc
  reference in the command palette).
- Removed ~30 dead `pub` functions, types, and consts across editor, lsp,
  stacker, terminal, git, theme_store, sketch, diagnostics, keybindings, and
  session (e.g. `EditorKeyChord` + its always-`None` `pending_key_chord` field,
  dead terminal-search helpers in `grid.rs`, unused `perf.rs` consts, the
  `session::Rect` struct).
- Cascade-removed two now-orphaned, unwired subsystems: the git-gutter LCS diff
  engine (`GitGutter::load` ran but its hunks were never rendered and recompute
  had no callers) and the git commit-detail parser/model (`git/detail.rs`).
  Both were on the "Leftovers roadmap" — deleted as unreachable, restorable
  from history if their wiring is meant to be finished.
- Intentionally kept future API surface per review decision: LSP async requests
  (`workspace_symbols`/`inlay_hints`/`code_lens` + `_async` siblings,
  `did_change_async`, health helpers) and `theme_store` save/load/delete.
- Two items the audit flagged as dead turned out to have real callers and were
  kept: `terminal/grid.rs::history_size` (used by the performance-budget
  integration test) and `perf.rs::LSP_CHANGE_LINE_LIMIT` (used by
  `live_lsp_enabled`).

**Modularization (zero behavior change, mechanical extractions)**
- `gpui_workspace/appearances.rs` (2236 LOC) split into `appearances/`:
  `shader_palettes`, `error_log`, `editor_section`, `terminal_section`,
  `widgets`.
- `gpui_sketch.rs` (1782 LOC) split into a new `gpui_sketch/` dir: `paint`,
  `canvas_element`, `render`.
- `gpui_stacker.rs` (1099 LOC): extracted the self-contained `StackerTextInput`
  widget into `gpui_stacker/text_input`.
- `stacker/cli.rs` (1202 LOC): moved the ~485-line inline test module into
  `stacker/cli/tests`.
- `gpui_editor/render.rs` split into `render/{overlays,markdown,status_bar}`.
- `gpui_stacker/render.rs` split into `render/{cli_help,toolbar}`.
- Deferred (HARD — large stateful methods on central structs):
  `gpui_workspace.rs`, `gpui_workspace/project.rs`, `gpui_editor/lsp.rs`,
  `effects/host.rs`.
