## Codebase Orientation

Reviewed the current shape of the repository and confirmed `llnzy` is a native Rust desktop app built around:

- `winit` for the app window and event loop
- `wgpu` for rendering
- `glyphon` for terminal/editor text rendering
- `egui` for app chrome and UI surfaces
- `portable-pty` plus `alacritty_terminal` for shell sessions
- `tree-sitter` and LSP support for the code editor

Key architecture notes captured:

- `src/main.rs` owns the app controller and runtime event loop.
- `src/runtime/` contains extracted app behavior for commands, tabs, layout, terminal handling, window state, and session restore.
- `src/ui/` owns egui surfaces such as tab bar, sidebar, Stacker, Sketch, settings, and editor/explorer views.
- `src/renderer/` owns GPU rendering, text, effects, backgrounds, cursor rendering, bloom, CRT, and particles.

One mismatch noted during orientation:

- `workspace_store` still contains session snapshot persistence, but current `runtime/session_restore.rs` clears the last session instead of restoring it.

## Stacker Formatting Controls

Added formatting controls above the prompt editor.

New controls:

- `B` button for Markdown bold
- unordered list button
- numbered list button
- `-` and `+` buttons for making the prompt editor text smaller or bigger
- `Import` button for importing prompt JSON from the local import file

Bold behavior:

- With selected text, wraps the selection as `**selected text**`
- With no selection, inserts `****` and places the caret between the markers

List button behavior:

- Unordered list button prefixes selected lines with `- `
- Numbered list button prefixes selected lines with `1. `, `2. `, etc.
- On an empty prompt, list buttons insert the starter marker

Font size behavior:

- The prompt editor text starts at `16px`.
- `-` decreases only the Stacker prompt editor text size.
- `+` increases only the Stacker prompt editor text size.
- Toolbar text remains compact and separate from the prompt editor font size.

## Automatic List Continuation

Added Enter-key list continuation inside the Stacker prompt editor.

Supported unordered markers:

- `- item`
- `* item`
- `+ item`

Supported ordered markers:

- `1. item`
- `1) item`

Behavior:

- Pressing Enter after a non-empty list item creates the next item.
- Numbered lists increment automatically.
- Pressing Enter on an empty list item exits the list.

Added focused helper tests for list continuation and list-button behavior.

## Prompt Queue Bank

Added a first-pass queue bank to the upper Stacker saved-prompts section.

Changes:

- Split the upper prompt area into two panes.
- Left pane is a small queue bank.
- Right pane is a scrollable saved-prompt list.
- Each saved prompt row now has an `Add to queue` button.
- Users can add up to five different prompts to the queue.
- Already queued prompts show a disabled `Queued` state.
- Queue entries render uppercase.
- Queue entries use green text to distinguish them from the saved-prompt list.
- Queue state is kept in `StackerUiState`.
- Queue cleanup removes duplicate, stale, or over-limit entries.
- Removed the saved-prompts search bar; the right pane now always shows the full scrollable prompt list.
- Doubled the target queue-bank width so queued prompts have more horizontal room.
- Queued prompts now surface in the footer when a terminal tab is active.
- Footer queue chips are labeled `A` through `E`.
- Footer queue chips use green text on a black background.
- Footer queue chips show the first three words of the queued prompt followed by an ellipsis.
- Footer queue chip labels use a colon after the letter, for example `A: Deploy preview build...`.
- Clicking a footer queue chip copies the full queued prompt text to the clipboard.
- The copied prompt preserves Markdown syntax such as `**bold**`, `- unordered list`, and `1. numbered list` markers.

Added tests for queue cleanup:

- blank queued prompt snapshots are removed
- duplicate queued prompt snapshots are removed
- queue length is capped at five

## Joined Tabs And Terminal Concurrency

Replaced the old split-panel direction with a joined-tab workflow.

Changes:

- Removed the old split-panel command path and split-terminal renderer path.
- Added a joined-tab state model that pairs the active tab with another existing tab.
- Added `Create Double Tab` to the tab context menu when right-clicking a non-active tab.
- Added `Separate Tabs` to the tab context menu when right-clicking either joined tab.
- Rendered joined tabs evenly across the content area with a fixed center divider.
- Supported joined rendering for Terminal, CodeFile, Home, Stacker, Sketch, Appearances, and Settings.
- Added pane-safe rendering entry points for Home, Appearances, and Settings as an initial step toward making every page responsive inside joined panes.
- Added click-to-activate behavior for joined terminal panes so users can click either shell and type into that shell.
- Adjusted terminal mouse/grid mapping so clicks, selection, and mouse reporting are relative to the active half-pane.
- Clamped narrow-pane widget widths in Stacker, editor search, project search, workspace symbols, and command palette to avoid egui negative-size crashes.

Terminal concurrency fix:

- Investigated a crash while running two busy CLI apps in joined terminal panes.
- Confirmed both PTY sessions were processing independently, but the GPU text renderer shared one glyphon terminal renderer across panes.
- Fixed the `glyphon vertices` destroyed-buffer crash by giving terminal tabs independent glyph renderers keyed by tab/cache id.
- Retained per-tab text caches/renderers only for currently visible terminal panes.
- Verified the joined-terminal path with terminal and tab-focused tests.
