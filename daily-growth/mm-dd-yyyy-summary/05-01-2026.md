## Codebase Orientation

Reviewed the current shape of the repository and confirmed `llnzy` is a native Rust desktop app built around:

- `winit` for the app window and event loop
- `wgpu` for rendering
- `glyphon` for terminal/editor text rendering
- `egui` for app chrome and UI surfaces
- `portable-pty` plus `alacritty_terminal` for shell sessions
- `tree-sitter` and LSP support for the code editor

Key architecture notes captured:

- `src/main.rs` owns the app controller and runtime event loop.
- `src/runtime/` contains extracted app behavior for commands, tabs, layout, terminal handling, window state, and session restore.
- `src/ui/` owns egui surfaces such as tab bar, sidebar, Stacker, Sketch, settings, and editor/explorer views.
- `src/renderer/` owns GPU rendering, text, effects, backgrounds, cursor rendering, bloom, CRT, and particles.

One mismatch noted during orientation:

- `workspace_store` still contains session snapshot persistence, but current `runtime/session_restore.rs` clears the last session instead of restoring it.

## Sidebar File Drag And Drop

Added the first filesystem-backed drag-and-drop move flow for the project sidebar.

Completed behavior:

- Files in the sidebar can be picked up with a drag gesture.
- Folder rows become drop targets for dragged sidebar files.
- A compact ghost label follows the cursor while a file is being dragged.
- Valid folder drop targets render a green outline while hovered.
- Dropping a file onto a folder emits a typed app drag/drop command.
- The runtime moves the file with `std::fs::rename`, so the change is reflected by macOS/Finder.
- Folder dragging is intentionally not enabled yet; this pass only supports moving files into folders.

Runtime safeguards:

- Rejects empty moves.
- Rejects folder sources.
- Rejects non-folder drop targets.
- Rejects same-parent drops.
- Rejects destination filename conflicts.
- Blocks moving an open file when that buffer has unsaved edits.
- Remaps clean open buffers and code-file tabs from the old path to the new path after a successful move.
- Updates file watching, LSP document state, syntax reparse state, and git gutter state for remapped open files.
- Refreshes the explorer tree after successful moves.

Architecture:

- Added `DragDropCommand::MoveFilesToFolder`.
- Added a small file-move planning helper in the app drag/drop layer.
- Kept sidebar rendering responsible for UI gestures and visual feedback only.
- Kept filesystem mutation and open-buffer remapping in the runtime command layer.
